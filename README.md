# ⌨️ StepTyper — Посимвольный ввод кода в Visual Studio

> 🎯 Нажмите любую клавишу → в редакторе появится **ровно один следующий символ** из вашего файла.  
> 🔒 Ваше нажатие **не дублируется** в Visual Studio.

---

## 📦 Что внутри
Всего 2 файла. Никаких установок и настроек.
- `AutoTyper.exe` — программа
- `PythonApplication1.txt` — файл с вашим кодом (откройте и вставьте свой текст)

---

## 🚀 Как пользоваться

1. **Подготовьте код**  
   Откройте `code.txt`, вставьте ваш код, сохраните (`Ctrl+S`).  
   *(Программа также автоматически понимает `.py`, `.cs`, `.js`, `.cpp` и другие расширения)*

2. **Запустите программу**  
   Нажмите правой кнопкой мыши на `AutoTyper.exe` → **Запуск от имени администратора**.  
   ⚠️ Без прав администратора блокировка нажатий не работает.

3. **Подготовьте Visual Studio**  
   - Откройте проект, кликните курсором в нужное место редактора.
   - Переключите раскладку клавиатуры на **EN-US** (обязательно для `{}`, `@`, `#`, `[]` и т.д.).

4. **Начинайте ввод**  
   Нажимайте любую клавишу на клавиатуре. В VS появится следующий символ из файла.  
   Нажимайте в своём темпе — программа ждёт каждое нажатие.

---

## 🎮 Управление

| Клавиша | Действие |
|---------|----------|
| `Любая клавиша` | Ввести следующий символ |
| `F1` | Сбросить счётчик (начать сначала) |
| `F2` | Пауза / Продолжить ввод |
| `Esc` | Закрыть программу |

---

## ⚠️ Важно знать

- 📁 Файл с кодом **должен лежать рядом** с `AutoTyper.exe`.
- 🌐 Всегда используйте раскладку **EN-US** перед запуском.
- 🛡️ Антивирус может ругаться на `.exe` из-за эмуляции клавиатуры. Это стандартное ложное срабатывание. Добавьте папку в исключения.
- ⚡ Если Visual Studio подтормаживает при вводе, временно отключите IntelliSense: `Ctrl + Shift + Space` или в настройках редактора.

---

## 🆘 Не работает?

| Проблема | Решение |
|----------|---------|
| Программа закрывается сразу | Запустите **от имени администратора** |
| Вводятся странные символы (`б` вместо `{`) | Переключите раскладку на **EN-US** |
| Пишет "Файл не найден" | Убедитесь, что `code.txt` лежит **в той же папке**, что и `.exe` |
| Нажатая клавиша всё равно появляется в VS | Проверьте запуск от админа. Без него подавление клавиш в Windows отключено |
